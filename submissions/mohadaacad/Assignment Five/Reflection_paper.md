# Understanding Random Forest

Random Forest is an ensemble machine learning algorithm that combines multiple Decision Trees. Instead of relying on one tree, the algorithm builds many trees using different subsets of the training data and features.

For classification tasks, each tree votes for a class label. The class that receives the majority of votes becomes the final prediction. This voting process helps reduce errors and improves overall performance.

One of the biggest advantages of Random Forest is that it can handle complex relationships in data while reducing overfitting. Compared to a single Decision Tree, Random Forest is usually more accurate and robust.

# Other Algorithms (Decision Tree)

For the additional classifier, I selected **Decision Tree Classification**.

A Decision Tree works by repeatedly splitting the dataset into smaller groups based on feature values. Each split creates a decision rule, and the final leaf node determines the predicted class.

## Advantages

- Easy to understand and visualize
- Highly interpretable
- Handles both numerical and categorical data

## Limitations

- Can easily overfit the training data if it becomes too complex
- Small changes in the dataset may produce different tree structures

From my research and implementation, I learned that Decision Trees provide clear decision rules and can handle both numerical and categorical data. However, their performance is often improved when combined into an ensemble such as Random Forest.

When comparing metrics, the Decision Tree model generally performed better than Logistic Regression in capturing non-linear relationships, but Random Forest often achieved the best overall results because it reduces the weaknesses of individual trees.

# Metrics Discussion

The three models were evaluated using:

- Accuracy
- Precision
- Recall
- F1-Score

The model with the highest **Accuracy** correctly classified the largest percentage of loan applications.

- **Precision** measured how many approved loans were actually correct.
- **Recall** measured how many truly approved applications were successfully identified.
- **F1-Score** provided a balanced measure of Precision and Recall.

In most cases, **Random Forest** achieved the strongest combination of metrics. A higher Accuracy and F1-Score indicate that the model was able to make correct predictions while maintaining a good balance between false positives and false negatives.

Logistic Regression served as a useful baseline because it was simple and easy to interpret. Decision Tree improved flexibility by modeling more complex patterns, while Random Forest delivered stronger performance through ensemble learning.
